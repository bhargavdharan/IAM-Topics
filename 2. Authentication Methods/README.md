# 2. Authentication Methods and Password Security

## What Is Authentication?

**Authentication** is the process of proving that an identity is genuine. When you enter a username and password, swipe a badge, or scan your fingerprint, you are authenticating.

Authentication relies on **factors** — different categories of proof. The strength of authentication depends on how many distinct factors you use and how difficult each factor is to forge or steal.

---

## Why Learn This?

Passwords are the most common authentication method and also the most commonly compromised. Understanding how authentication works — and how it fails — is essential whether you are:
- Building applications that handle logins
- Choosing security policies for an organization
- Protecting your own personal accounts
- Investigating security incidents

---

## Core Concepts

### Authentication Factors

A factor is a category of evidence. Using multiple factors from different categories is the foundation of strong authentication.

| Factor | Category | Examples | Strength |
|--------|----------|----------|----------|
| Something you **know** | Knowledge | Password, PIN, security question | Weak — can be guessed or phished |
| Something you **have** | Possession | Phone, smart card, hardware key | Strong — requires physical access |
| Something you **are** | Inherence | Fingerprint, face, iris | Strong — hard to fake |
| Somewhere you **are** | Location | GPS, IP address | Weak — can be spoofed |
| Something you **do** | Behavior | Typing rhythm, mouse movements | Moderate — requires baselining |

**Key principle:** Two factors from the *same category* (password + security question) is not true multi-factor authentication. True MFA requires factors from *different* categories (password + phone, or fingerprint + hardware key).

### Password Security

Passwords persist because they are cheap, universally understood, and work on every device. But they are also the weakest link in most security architectures.

**How passwords fail:**
- **Phishing:** Fake login pages trick users into entering credentials
- **Data breaches:** Databases are stolen containing millions of passwords
- **Reuse:** People use the same password across multiple sites
- **Weak choices:** Common passwords like "123456" or "password" are still prevalent
- **Keyloggers:** Malware records keystrokes

**What makes a password strong:**
- Length is the most important factor. A 16-character passphrase is stronger than an 8-character complex password.
- Unpredictability matters more than complexity. `correct-horse-battery-staple` (from xkcd) is stronger than `Tr0ub4dor&3` because it is longer and easier to remember.
- Uniqueness per account prevents credential stuffing attacks.
- NIST guidelines (SP 800-63B) now recommend against forced periodic changes unless compromise is suspected, because frequent changes lead to predictable patterns.

### Modern Authentication Standards

| Standard | How It Works | Best For |
|----------|-------------|----------|
| **FIDO2 / WebAuthn** | Public key cryptography bound to the website domain | Passwordless login resistant to phishing |
| **Risk-Based Authentication** | Adjusts requirements based on signals like location, device, and behavior | Balancing security and user experience |
| **Biometrics** | Mathematical templates of fingerprints, faces, or iris patterns | Convenient mobile authentication |

## How It Works

### Password Hashing: The Critical Protection

When you create a password, the system must store something to verify future logins. **Storing the password itself is never acceptable.** If the database is breached, attackers gain every password immediately.

Instead, systems store a **hash** — a fixed-length fingerprint generated by a one-way mathematical function.

**Properties of a good hash:**
- Same input always produces the same output
- Different inputs produce drastically different outputs
- The original input cannot be recovered from the output

**Why simple hashing is not enough:**
Attackers build **rainbow tables** — pre-computed databases of common passwords and their hashes. If your password is "hello123", the attacker simply looks up its hash.

**Salting defeats rainbow tables:**
A **salt** is a random string unique to each password. The system hashes `password + salt` instead of just `password`. Even if two users have the same password, their hashes are completely different.

```python
# Simplified (do not use MD5 in production)
import hashlib, secrets

salt = secrets.token_hex(16)  # Random per user
hash_value = hashlib.sha256((password + salt).encode()).hexdigest()
```

**Adaptive hashing (bcrypt, Argon2):**
Modern algorithms intentionally take time to compute. Bcrypt has a **cost factor** that makes hashing slower as computers get faster. Argon2 is **memory-hard**, requiring significant RAM to compute, which defeats GPU-based cracking.

| Algorithm | Status | Why |
|-----------|--------|-----|
| MD5 | Broken | Trivial to crack; collisions demonstrated |
| SHA-1 | Deprecated | Collision attacks published |
| SHA-256 | Fast alone | Needs iteration + salting for passwords |
| bcrypt | Recommended | Adaptive cost factor |
| scrypt | Recommended | Memory-hard |
| Argon2 | Best practice | Winner of Password Hashing Competition |

### How Biometric Authentication Actually Works

Your fingerprint is **not stored as an image**. Images are too large, violate privacy, and are easy to fake with photographs.

Instead, biometric systems extract **minutiae points** — specific features like ridge endings and bifurcations — and store a mathematical template. When you scan your finger again, the system extracts features from the new scan and compares them to the stored template.

**Two key metrics:**
- **FAR (False Acceptance Rate):** The probability that the wrong person is accepted. Lower is more secure.
- **FRR (False Rejection Rate):** The probability that the right person is rejected. Lower is more convenient.

These metrics trade off against each other. A banking application prioritizes low FAR (security). A phone unlock prioritizes low FRR (convenience).

### How FIDO2 Prevents Phishing

FIDO2 uses public key cryptography:
1. **Registration:** Your device generates a unique key pair. The public key goes to the server. The private key **never leaves your device**.
2. **Authentication:** The server sends a challenge. Your device signs it with the private key. The server verifies the signature with the public key.

**Why this is phishing-resistant:** The private key is cryptographically bound to the website's domain. A fake "gmail.com" site cannot use your real gmail.com credentials because the keys do not match.

## Common Misconceptions

| Misconception | Reality |
|--------------|---------|
| "Complex passwords are always better" | Length beats complexity. Passphrases are stronger and easier to remember |
| "Biometrics are stored as images" | Only mathematical templates are stored; raw images are never kept |
| "MFA with SMS is as good as an app" | SMS is vulnerable to SIM swapping and interception; app-based TOTP is safer |
| "Hashing alone is enough" | Salting is mandatory; adaptive algorithms (bcrypt/Argon2) are recommended |
| "FIDO2 is only for enterprises" | Passkeys (consumer FIDO2) are now built into Apple, Google, and Microsoft accounts |

## How to Practice

1. **Inspect password storage**
   - Run `password_hash_demo.py` to see MD5 vs bcrypt vs Argon2 in action
   - Notice the computation time differences
   - Understand why slower is better for password hashing

2. **Evaluate your own passwords**
   - Run `password_policy_enforcer.py`
   - Test passwords you currently use (in the tool, not on real sites)
   - Understand entropy and why length matters most

3. **Observe risk-based authentication**
   - Log into your bank from a new device or location
   - Notice the extra steps required
   - Identify the risk signals: new device, new location, unusual time

4. **Set up a password manager**
   - If you do not already use one, install Bitwarden, 1Password, or KeePass
   - Generate unique 20-character passwords for every account
   - Enable MFA on your most important accounts
